# Contribuição para o Projeto

Bem-vindo! Siga estas diretrizes para colaborar de forma eficiente e manter a qualidade do projeto.

## Como colaborar

1. **Fork e Pull Request**
   - Faça um fork do repositório e crie uma branch para sua feature ou correção.
   - Envie um Pull Request (PR) para a branch `main`.
   - Descreva claramente o que foi alterado/adicionado.

2. **Padrão de Código**
   - Siga o padrão PEP8 para Python.
   - Use nomes de variáveis e funções descritivos (em inglês).
   - Adicione docstrings e comentários quando necessário.
   - Prefira funções puras e modularização.

3. **Testes**
   - Sempre que possível, adicione ou atualize testes para cobrir suas mudanças.
   - Rode todos os testes antes de abrir um PR.

4. **Commits**
   - Commits devem ser pequenos, claros e frequentes.
   - Use o padrão de mensagem de commit abaixo.

## Padrão de Mensagem de Commit

```
<tipo>(escopo): descrição breve

[Corpo opcional explicando o motivo da mudança]
```

### Tipos comuns
- feat: Nova funcionalidade
- fix: Correção de bug
- refactor: Refatoração de código
- docs: Mudanças na documentação
- test: Adição/correção de testes
- chore: Tarefas de manutenção

### Exemplos
- feat(api): adiciona endpoint de login
- fix(workflow): corrige bug no fluxo de aprovação
- docs: atualiza README com instruções de setup

## Revisão de Código
- PRs serão revisados por pelo menos um mantenedor.
- Responda comentários e faça ajustes solicitados.

## Outras dicas
- Sempre sincronize sua branch com a `main` antes de abrir um PR.
- Evite commits grandes e mistos (ex: feature + refactor juntos).
- Prefira inglês para código e comentários, português para documentação.

Obrigado por contribuir!
